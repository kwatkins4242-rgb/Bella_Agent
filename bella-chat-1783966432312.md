To hook up n8n to your project, your app needs to call the **production webhook URL** from wherever it starts a new “Bella” session.

Quick checklist:

1. **Use the production URL, not the test URL**
   - Test URL (only works when the editor is listening):  
     `http://localhost:5678/webhook-test/bella-session-start`
   - Production URL (works when the workflow is activated):  
     `http://localhost:5678/webhook/bella-session-start`
2. **Activate the workflow** in the n8n editor (toggle in the top-right).
3. **Make sure the Webhook node returns the MongoDB summaries**:
   - Response mode: **“Last Node”** (so the last node’s JSON is returned) **or** add a **Respond to Webhook** node.
   - The last node should output the summaries you want your app to receive.
4. **Call the webhook from your project** when a session starts.

Example call from a JavaScript/Node project:

```js
const response = await fetch("http://localhost:5678/webhook/bella-session-start", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ userId: "123", sessionId: "abc" }),
});

const context = await response.json();
// now inject `context` into your LLM prompt
```

Or Python:

```python
import requests

res = requests.post(
    "http://localhost:5678/webhook/bella-session-start",
    json={"userId": "123", "sessionId": "abc"}
)
context = res.json()
```

5. **If your app and n8n are not on the same machine**, `localhost` won’t work. Use:
   - The machine’s IP address, or
   - **ngrok** for local tunneling:  
     `ngrok http 5678` → gives you a public URL like `https://xxxx.ngrok-free.app/webhook/bella-session-start`
   - Or deploy n8n to a server / cloud.

6. **If your app is a browser frontend**, you may hit CORS. In that case it’s usually better to call n8n from your backend, not directly from the browser.
